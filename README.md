# InFynd Microservices Backend

Production-grade Node.js microservices deployed via Docker Compose with NGINX load balancing, MongoDB replica set, Kafka cluster, and Redis.

## Architecture

| Service | Replicas | Port | Description |
|---|---|---|---|
| NGINX | 1 | 80, 443 | Load balancer / reverse proxy |
| API Gateway | 2 | 3000 | Request routing & rate limiting |
| Auth Service | 2 | 3001 | Authentication & authorization |
| Notification Service | 2 | 3002 | Email & notification handling |
| MongoDB | 3 | 27017 | Replica set (primary + secondary + arbiter) |
| Redis | 1 | 6379 | Caching & sessions |
| Kafka | 3 | 9092-9094 | Event streaming cluster |
| Zookeeper | 1 | 2181 | Kafka coordination |

## Project Structure

```
├── services/
│   ├── api-gateway/          # API Gateway service + Dockerfile
│   ├── auth-service/         # Auth service + Dockerfile
│   └── notification-service/ # Notification service + Dockerfile
├── scripts/
│   ├── mongo-entrypoint.sh   # MongoDB container entrypoint
│   ├── mongo-init.sh         # MongoDB initialization (users, indexes)
│   ├── mongo-keyfile         # Replica set auth key (generated by deploy.sh)
│   ├── setup-mongodb-replica.sh  # rs.initiate() script
│   └── setup-kafka-topics.sh     # Kafka topic creation
├── nginx/
│   └── nginx.conf            # NGINX load balancer config
├── shared/
│   └── tenant-config/        # Multi-tenant configuration files
├── docker-compose.production.yml  # Production compose (all services)
├── deploy.sh                      # Full deployment script
├── .env                           # Active environment variables
└── .env.production                # Production env template
```

## Deployment

### Prerequisites
- Docker and Docker Compose installed

### Deploy

```bash
chmod +x deploy.sh
./deploy.sh
```

The deploy script handles the full startup sequence: Zookeeper → MongoDB replica set init → Redis → Kafka cluster → Application services → NGINX.

### Useful Commands

```bash
# View logs
docker compose -f docker-compose.production.yml logs -f

# Stop all services
docker compose -f docker-compose.production.yml down

# Restart a specific service
docker compose -f docker-compose.production.yml restart <service>
```

### API Endpoints

- **Health Check**: `GET http://localhost/health`
- **Auth API**: `http://localhost/auth/*`
